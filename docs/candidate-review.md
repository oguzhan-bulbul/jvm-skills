# Candidate Review — skill-scout pile, strict recheck (2026-07-03)

Adversarial re-judging of **all 483 candidates** in `skill-scout/db/skill_files.csv`
(349 `found`, 134 `needs_review`) against **`docs/skill-approval-rulebook.md`**, deduplicated
within the pile and against the 31 existing listings. Every verdict below is grounded in the
**actual SKILL.md body** (or skill-unit: SKILL.md + co-located references) fetched from GitHub —
never the scanner's CSV reasoning, which this pass repeatedly disproved in both directions.

**Method.** 18 cluster judges (one per author/topic group) ran the rulebook's §7 decision
procedure; ~140 file bodies were read in full, large collections were stratified-sampled with
tree-size censuses, and **~70 API claims were verified against real library sources** (Spring
Boot 4, Quarkus, Micronaut, Kotlin 2.2/2.3, FXGL, Koog 1.0.0, PDFBox/POI, Vaadin 25, Firebase,
qs-kotlin, OKafka, langchain4j-community, jstall, jpackage/notarytool, ~15 JEPs). The 189
candidates that also carry a later-dated rejection in `rejected.csv` were treated as
presumptively rejected and spot-verified; a handful were explicitly **overturned** where the repo
had changed or the original verdict misapplied a rule.
